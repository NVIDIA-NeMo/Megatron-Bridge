### #2674 — How to parse nargs='+' args in bridge configs

| Field | Value |
|---|---|
| **Type** | `support` (already set) |
| **Area** | `area:misc` (already set) |
| **State** | `needs-triage` → `needs-follow-up` |
| **Author** | cryoco (external, `community-request` already set) |
| **Label changes** | remove `needs-triage`, add `needs-follow-up` |

**Proposed response:**

> Bridge uses OmegaConf + Hydra overrides, not argparse `nargs='+'`. For list-type config values, pass them as a Python list on the CLI:
>
> ```bash
> uv run torchrun --nproc_per_node=8 scripts/training/run_recipe.py \
>     --recipe qwen3_8b_pretrain_config \
>     model.cuda_graph_scope="[attn,moe_preprocess,moe_router]"
> ```
>
> Or in a YAML config file:
>
> ```yaml
> model:
>   cuda_graph_scope:
>     - attn
>     - moe_preprocess
>     - moe_router
> ```
>
> Since recipes already define `cuda_graph_scope` with a default value, a plain `key=value` override is all you need. If you ever need to **add** a key that doesn't exist in the recipe, use the Hydra `+` prefix:
>
> ```bash
> +model.some_new_field=42
> ```
>
> In most cases you won't need `+` — the recipes set sensible defaults for all standard fields.
