### Performance

#### Muon Optimizer
- **Source code**: `src/megatron/bridge/training/optim.py`, `src/megatron/bridge/recipes/utils/optimizer_utils.py`
- **Verification steps**:
  1. Check optimizer config in recipes (e.g., `src/megatron/bridge/recipes/kimi/kimi_k2.py`)
  2. Start a training run with Muon optimizer:
     ```bash
     uv run -m torch.distributed.run --nproc_per_node=8 scripts/training/run_recipe.py \
         --recipe qwen3_30b_a3b_pretrain_config \
         optimizer.optimizer=dist_muon \
         optimizer.use_distributed_optimizer=false \
         optimizer.muon_momentum=0.95 \
         optimizer.muon_use_nesterov=true \
         optimizer.muon_scale_mode=spectral \
         optimizer.muon_fp32_matmul_prec=medium \
         optimizer.muon_num_ns_steps=5 \
         optimizer.muon_tp_mode=blockwise \
         ddp.overlap_param_gather=false \
         ddp.use_distributed_optimizer=false \
         scheduler.start_weight_decay=0.033 \
         scheduler.end_weight_decay=0.033
     ```
  3. Verify Muon optimizer initializes correctly in training logs

#### NVFP4

  3. Compare memory usage and performance across precision formats
